#+title: hyprland-zen native host

* Components

- ~hyprland-zen-native-host~
  - single executable with two modes:
    - default: Native Messaging adapter launched by Zen/Firefox
    - ~--line-stdio~: line JSON bridge used by Emacs (~hyprland-zen.el~)

The browser-side default mode and Emacs-side ~--line-stdio~ mode communicate
through one unix socket.

~--line-stdio~ mode emits additional runtime diagnostics for Emacs:

- ~bridge-state~: browser bridge connected/disconnected transitions
- ~queued~: operation reached line-stdio host while browser side unavailable
- ~error~: immediate forwarding/decoding failures

Both scripts use this socket path:

- ~HYPRLAND_ZEN_SOCKET~ env var when set
- else ~$XDG_RUNTIME_DIR/hyprland-zen-bridge.sock~
- else ~/tmp/hyprland-zen-bridge-<uid>.sock~

* Install notes

1. Put ~hyprland-zen-native-host~ on your PATH (or use absolute path)
2. Copy ~hyprland_zen_bridge.json~ to your browser's NativeMessagingHosts dir
3. In that JSON file, set ~path~ to absolute path of ~hyprland-zen-native-host~
4. Ensure ~allowed_extensions~ includes extension id from ~manifest.json~

* Emacs side

Default ~hyprland-zen-host-command~ is ~("hyprland-zen-native-host" "--line-stdio")~.
If not on PATH, you can set:

#+begin_src emacs-lisp
(setq hyprland-zen-host-command '("/absolute/path/to/hyprland-zen-native-host" "--line-stdio"))
#+end_src
