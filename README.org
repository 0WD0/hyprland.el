#+title: hyprland.el

* hyprland.el

~hyprland.el~ brings Hyprland window management into Emacs.

- Snapshot + socket2 sync for Hyprland windows
- Reconcile policy to keep state converged under noisy events
- Consult-based window switch with static preview
- Ibuffer-native workflow via mirrored window buffers
- Experimental Zen tab bridge over native-host JSON protocol

* Status

Current version: ~0.1.0~ (early but usable).

* Requirements

- Emacs 29.1+
- Hyprland session (with ~HYPRLAND_INSTANCE_SIGNATURE~ in Emacs env)
- ~hyprctl~ in PATH
- ~grim~ in PATH (for static preview)
- Optional: ~hyprland-toplevel-snap~ helper (for workspace/occlusion-robust preview)
- [[https://github.com/minad/consult][consult]] (for ~hyprland-window-switch~)
- Optional: ~hyprland-zen-native-host~ executable (for ~hyprland-zen-*~ commands)
- Optional: Zen/Firefox WebExtension under ~browser/zen-extension/~

* Installation

#+begin_src emacs-lisp
(add-to-list 'load-path "/home/_WD_/Dev/hyprland.el")

#+end_src

** Eask dependencies

#+begin_src bash

#+end_src

* Basic usage

#+begin_src emacs-lisp
(require 'hyprland)
(hyprland-sync-start)
#+end_src

Core commands:

- ~M-x hyprland-window-switch~: consult picker + static preview + jump
- ~M-x hyprland-refresh~: force full snapshot
- ~M-x hyprland-jump~: focus a window by address
- ~M-x hyprland-close~: close a window by address
- ~M-x hyprland-tag~: apply ~+foo/-foo/foo~ tag op
- ~M-x hyprland-ibuffer-doctor~: repair ibuffer profile/index drift
- ~M-x hyprland-zen-start~: start Zen tab host bridge
- ~M-x hyprland-zen-tab-switch~: choose and activate a Zen tab
- ~M-x hyprland-zen-workspace-switch~: choose and activate a Zen workspace
- ~M-x hyprland-sync-stop~: stop socket/timers

* Toplevel helper (recommended)

Build helper with ~hyprwayland-scanner~:

#+begin_src bash
make -C tools
#+end_src

By default, preview backend is ~auto~:

- try ~hyprland-toplevel-snap~ first
- fallback to ~grim~ when helper is unavailable/fails

If helper is not installed in PATH, this repo's local path
~tools/hyprland-toplevel-snap~ is auto-detected.

* Ibuffer workflow (recommended)

Enable mirror mode once:

#+begin_src emacs-lisp
(hyprland-ibuffer-mirror-mode 1)
#+end_src

Open Hyprland-only ibuffer view:

- ~M-x hyprland-ibuffer-open~ (native saved filter-group profile)

You normally do not need to run install commands manually:
~hyprland-ibuffer-open~ auto-installs/updates the Hyprland saved filter profile.

This view uses Hyprland-focused columns:

- Workspace, Class, Title, Address, PID
- Killing a Hyprland mirror buffer closes its corresponding Hyprland window
- During completion preview, mirror buffers render in preview window; confirm/open jumps to real Hyprland window

Inside this ibuffer view, existing ibuffer commands are remapped:

- ~RET~, ~f~, ~e~, ~o~, ~M-o~ -> jump to Hyprland window
- ~k~ -> close marked Hyprland windows (or current row if nothing marked)

No extra custom keybindings are required.

* Zen tab bridge (experimental)

This is the first integration slice for browser-tab workflows.

Plugin scaffold now exists in this repo:

- ~browser/zen-extension/manifest.json~
- ~browser/zen-extension/background.js~
- ~browser/native-host/hyprland-zen-native-host~ (single executable for browser native mode + Emacs line-stdio mode)
- ~browser/native-host/hyprland_zen_bridge.json~ (native host manifest template)

It runs on Firefox-compatible APIs, so it works on Zen and plain Firefox.
Workspace is currently derived as ~windowId + cookieStoreId~.
For future Zen-only workspace semantics, we can add an optional Zen-specific
provider while keeping this fallback path.

Testing path:

1. Load temporary extension from ~browser/zen-extension/manifest.json~
2. Set executable permissions (once):

#+begin_src bash
chmod +x browser/native-host/hyprland-zen-native-host
#+end_src

3. Install native host manifest from ~browser/native-host/hyprland_zen_bridge.json~ and set its ~path~ to absolute ~hyprland-zen-native-host~
4. In Emacs run:

#+begin_src emacs-lisp
(hyprland-zen-start)
(hyprland-zen-refresh)
(hyprland-zen-refresh-workspaces)
#+end_src

Without native host wiring, extension can still be smoke-tested via Browser
Toolbox logs (~[hyprland-zen-extension]~).

Start bridge process:

#+begin_src emacs-lisp
(hyprland-zen-start)
#+end_src

Use tab commands:

- ~M-x hyprland-zen-refresh~ (request full snapshot)
- ~M-x hyprland-zen-refresh-workspaces~ (request workspace snapshot)
- ~M-x hyprland-zen-tab-switch~ (activate selected tab and focus its browser window)
- ~M-x hyprland-zen-tab-close~ (close selected tab)
- ~M-x hyprland-zen-workspace-switch~ (activate selected workspace)
- ~M-x hyprland-zen-open-url~ (open URL in Zen)

Tab switch now supports preview during candidate navigation:

- selection UI uses Consult state callbacks
- preview requests ~capture-tab~ from host/extension
- preview image is shown in shared hyprland preview window before confirm

Host protocol (line-delimited JSON, one object per line):

- inbound tab events: ~snapshot~, ~upsert~, ~remove~
- inbound workspace events: ~workspace_snapshot~, ~workspace-upsert~, ~workspace-remove~
- outbound ops: ~list-tabs~, ~list-workspaces~, ~activate-tab~, ~close-tab~, ~activate-workspace~, ~open-url~

Limitations from current Zen WebExtension surface:

- Zen internal workspace UUID and synced-window identity are not exposed to WebExtensions.
- Current mapping uses public fields (~windowId~, ~cookieStoreId~) and emits ~sync_group~ as a heuristic (~container:<cookieStoreId>~).

* Customization

Important options:

- ~hyprland-debug~: verbose debug messages
- ~hyprland-hyprctl-executable~: path to ~hyprctl~
- ~hyprland-grim-executable~: path to ~grim~
- ~hyprland-preview-sensitive-regexp~: skip screenshot preview for matching windows
- ~hyprland-preview-cache-mode~: ~memory-only~ (default) or ~disk~
- ~hyprland-preview-cache-directory~: cache directory when using disk mode
- ~hyprland-preview-cache-max-entries~ / ~hyprland-preview-cache-max-bytes~: in-memory preview cache bounds
- ~hyprland-preview-capture-backend~: ~auto~ (default), ~toplevel~, or ~grim~
- ~hyprland-preview-toplevel-helper-executable~: helper executable name/path
- ~hyprland-preview-focus-for-accurate-capture~: default ~nil~ (avoid focus flicker); set ~t~ only if you prefer aggressive grim fallback accuracy
- ~hyprland-consult-preview-key~: Consult preview trigger (e.g. ~any~, ~(:debounce 0.3 any)~, ~"M-."~, or ~nil~)
- ~hyprland-consult-preview-display~: ~current-window~ (default, Consult-style) or ~side-window~
- ~hyprland-ibuffer-filter-group-name~: display name of Hyprland filter group
- ~hyprland-ibuffer-filter-group-position~: place group at beginning or end
- ~hyprland-ibuffer-saved-filter-group-profile~: ibuffer saved-group profile name used by native open command
- ~hyprland-ibuffer-intercept-buffer-open~: redirect mirror-buffer open actions into ~hyprland-jump~
- ~hyprland-zen-host-command~: command list used to launch Zen bridge process
- ~hyprland-zen-auto-refresh-on-start~: request tab snapshot when bridge starts
- ~hyprland-zen-preview-key~: Consult preview trigger for tab switch
- ~hyprland-zen-jump-to-window-on-tab-switch~: best-effort Hyprland jump on tab activate

Reconcile defaults (already tuned):

- debounce: 120ms
- fast min interval: 400ms
- fast burst: 6 per 10s
- periodic full reconcile: 8s

* Troubleshooting

** "Missing XDG_RUNTIME_DIR or HYPRLAND_INSTANCE_SIGNATURE"

Your Emacs process does not have Hyprland session env vars.
This often happens with daemon/service launches.

** Timer error about "Invalid time frequency"

Use latest ~hyprland-reconcile.el~ from this repo. That compatibility issue
is fixed in current code.

** Consult state arity error

Reload updated package files:

#+begin_src emacs-lisp
(unload-feature 'hyprland-consult t)
(unload-feature 'hyprland t)
(require 'hyprland)
#+end_src

** Ibuffer commands not remapped

Ensure you opened the special view via ~hyprland-ibuffer-open~ and that
~hyprland-ibuffer-view-mode~ is enabled in that ibuffer buffer.

** Ibuffer filter-group errors

If ibuffer reports an undefined Hyprland filter, reinstall the saved profile:

#+begin_src emacs-lisp
(hyprland-ibuffer-install-saved-filter-group)
(hyprland-ibuffer-open)
#+end_src

You can also run a one-shot repair + diagnostics command:

#+begin_src emacs-lisp
(hyprland-ibuffer-doctor)
#+end_src
